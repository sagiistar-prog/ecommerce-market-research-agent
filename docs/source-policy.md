# Source Policy

This project uses a conservative source policy because it is a public portfolio demo. The repository should never require private source bundles, real client data, real partners, or real commercial records to demonstrate the workflow.

## Source Priority

| Priority | Source Type | Evidence Level | Safe Demo Use |
| --- | --- | --- | --- |
| 1 | Fictional product brief | E0 | Used |
| 2 | Synthetic competitor table | E0 | Used |
| 3 | Reviewed user-provided worksheet | E1 | Not used in Safe Demo |
| 4 | Reviewed public reference with citation log | E2 | Not used in Safe Demo |
| 5 | Agent-generated hypothesis | E4 | Allowed only when labeled |

## Evidence Rules

- E0 is for portfolio fixtures only.
- E1 and above require source-owner approval or citation logging before production use.
- E4 must be written as a hypothesis and converted into a test.
- The report must not blur the line between fixture facts and recommendations.

## Manual Review Required

Manual review is required before using output in public copy when it affects:

- Product claims.
- Compliance assumptions.
- Country-specific rules.
- Safety, health, performance, origin, warranty, or durability language.
- Pricing, shipping, return, and fulfillment expectations.

## Public Repository Boundary

The public demo should only include generalized, fictional, or synthetic examples. It should not reveal private operating paths, partner relationships, customer information, monetization design, product planning, or confidential project details from any real project.
